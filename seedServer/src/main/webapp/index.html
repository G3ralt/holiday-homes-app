Semester Seed - REST Backend

This is a backend for this matching React Front End

This Backend provides, out of the box:

  • Token Based Authentication
  • Ready to use with JPA. It ships with a User and a Role Entity class, meant for Authentication and Authorization
  • Ready to use with JAX-RS. Ships with a few dummy REST endpoints, see the rest package
  • Role based authorization, with two predefined user roles (with a simple sample REST-endpoint for each)
    • User
    • Admin
  • The seed is meant as an easy start, so when the option makeTestUser is set to true in web.xml, it create the following role and users:
    • ROLE: User
    • ROLE: Admin
    • User: username: user, password: test
    • admin: username: admin, password: test
    • user_admin: username: user_admin, password: test
  • Passwords are hashed and salted using PasswordStorage.js from here
  • Does not include a persistence.xml file (this file is git-ignored).
    You must setup your own local database, and use the NetBeans wizard to create a persistence.xml file.
    The name of this persistence unit MUST BE: pu_development
  • Built in filters to allow CORS (If you are hosting backend and frontend on the same URL) you can disable CORS by commenting out the @Provider anotations in the the two CORS-filters (see the cors package
